# Safety and Privacy

This repository is designed for public, generic, and fictional examples.

Do not commit sensitive information.

## Never Commit

- Credentials
- API keys
- OAuth tokens
- Company data
- Personal data
- Contracts
- Unreleased business plans
- Private metrics
- Internal file paths
- Internal logs
- Private storage links
- Customer records
- Confidential meeting notes

## Safer Alternatives

- Use fictional examples.
- Use public documentation.
- Summarize patterns instead of copying raw content.
- Replace names, numbers, and paths with generic placeholders.
- Keep private source files outside the repository.

## Review Before Commit

Before committing, search for:

- Secret-like strings
- Real organization names
- Personal names
- Private paths
- Raw exports
- Credentials or token files

When in doubt, leave it out.
