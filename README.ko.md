# AI Work OS Starter Kit

AI 에이전트를 단발성 채팅 도구가 아니라 장기적인 업무 운영 체계로 활용하기 위한 가벼운 스타터 키트입니다.

ChatGPT, Codex, Claude, Gemini 같은 도구가 누적 맥락을 더 잘 활용할 수 있도록 출처, 결정, 프롬프트, 루틴, 유지보수 기록을 정리하는 기본 구조를 제공합니다.

English README: [README.md](README.md)

## 이 프로젝트가 해결하는 문제

AI 도구는 이전 맥락을 쉽게 잊습니다. 팀의 결정은 여러 채팅과 문서에 흩어지고, 한 번 만든 프롬프트도 다시 쓰기 어려워집니다.

이 스타터 키트는 AI 업무에 간단한 운영 구조를 더합니다. 출처 인덱스, 프로젝트 인덱스, 핸드오프 프롬프트, 리뷰 루틴, 유지보수 로그를 통해 AI 보조 업무를 다시 이어가기 쉽고, 검토하기 쉽고, 개선하기 쉬운 형태로 관리합니다.

## 이 프로젝트가 맞는 것 / 아닌 것

이 프로젝트는 다음에 가깝습니다.

- Markdown 기반의 가벼운 업무 운영 구조
- AI 보조 업무의 연속성을 위한 스타터 키트
- PM, 분석가, 운영 담당자, 비개발자도 사용할 수 있는 실무 템플릿

이 프로젝트는 다음이 아닙니다.

- 자동화 플랫폼
- 비공개 데이터 보관소
- 사람의 검토를 대체하는 시스템
- 완성형 지식 관리 제품

## 이런 경우에 사용하세요

- AI 도구를 반복적으로 쓰지만 세션마다 맥락이 끊깁니다
- AI 에이전트에게 더 안전하게 작업을 넘기고 싶습니다
- 출처, 결정, 프롬프트를 가볍게 추적하고 싶습니다
- 앱이나 DB를 만들기 전에 Markdown 기반 운영 구조부터 만들고 싶습니다

## 누구를 위한 프로젝트인가

- 리서치, 출시, 보고서, 의사결정을 조율하는 PM
- 반복 가능한 근거 관리와 보고서 흐름이 필요한 분석가
- 일간/주간 AI 업무 루틴을 만들고 싶은 운영 담당자
- 개발자는 아니지만 AI 도구를 체계적으로 쓰고 싶은 실무자
- 무거운 시스템 없이 팀의 AI 작업 습관을 만들고 싶은 소규모 팀

## 핵심 아이디어

AI 에이전트는 맥락, 출처, 결정, 루틴이 구조화되어 있을 때 더 잘 작동합니다.

매번 흩어진 정보를 채팅창에 다시 붙여넣는 대신, 이 키트는 다음을 정리하도록 돕습니다.

- 출처 인덱스: 어떤 정보가 어디에서 왔는지
- 프로젝트 인덱스: 어떤 일이 진행 중이고 어떤 결정이 있었는지
- 스킬 인덱스: 재사용 가능한 프롬프트, 업무 흐름, 운영 패턴
- 핸드오프 프롬프트: AI 에이전트에게 줄 명확한 작업 지시
- 유지보수 로그: 무엇이 언제 왜 바뀌었는지

목표는 과장된 자동화가 아니라, 업무 맥락을 안전하고 반복 가능하게 쌓는 것입니다.

## 10분 빠른 시작

1. `docs/safety_and_privacy.md`를 먼저 읽습니다.
2. 민감하지 않은 연습용 프로젝트나 허구 프로젝트를 하나 고릅니다.
3. `01_wiki/project_index.md`에 프로젝트를 한 줄 추가합니다.
4. `01_wiki/source_index.md`에 출처 참조를 추가합니다.
5. `03_templates/codex_handoff_template.md`를 복사해 AI 에이전트에게 전달합니다.
6. 작고 명확한 작업 하나를 요청합니다.
7. 결과물을 사람이 직접 검토합니다.
8. `02_routines/maintenance_log.md`에 변경 내용을 남깁니다.

## 추천 업무 흐름

1. 출처 수집
2. 인덱스 갱신
3. 핸드오프 프롬프트 작성
4. AI 에이전트 실행
5. 결과 검토
6. 유지보수 로그 갱신

이 흐름을 반복하면 맥락 재사용이 쉬워지고, AI 산출물 검토도 더 명확해집니다.

## 보안/프라이버시 주의

이 저장소에는 민감 정보나 비공개 정보를 커밋하지 마세요.

API 키, OAuth 토큰, 회사 데이터, 개인정보, 계약서, 미공개 사업 계획, 비공개 지표, 내부 파일 경로, 내부 로그를 절대 포함하지 마세요. 예시는 모두 허구이거나 공개 가능한 정보만 사용해야 합니다.

## 활용 예시

- 공개 출처 기반 주간 시장 스캔
- 허구의 제품 출시 보고서
- 개인 리서치 지식베이스
- 민감하지 않은 회의 노트 정리 흐름
- 공개 문서 유지보수
- 재사용 가능한 프롬프트와 리뷰 체크리스트 라이브러리

짧은 예시는 [docs/examples.md](docs/examples.md)에서 볼 수 있습니다.
